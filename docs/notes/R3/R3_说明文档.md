## 四、与其他成员的交互（重点）

### 4.1 R3 → R4（杨天阳）：提供 UI 组件 + 路由 + 缓存写入

R4 是"集成终端"，消费 R1/R2/R3。R3 给 R4 提供：

| R3 提供的接口 | R4 在哪里用 |
|--------------|------------|
| `PageLayout` | 首页、结果页外层容器 |
| `ScoreDisplay` | 结果页总评分区 |
| `StatusIcon` | 结果页 9 项检测列表 |
| `LevelTag` | 结果页仓库卡片 |
| `LoadingState` | 检测流程 loading 态 |
| `ErrorState` | 检测流程 4 种错误态（按 R1 的 `error.kind` 映射文案后传入 `error` 字段） |
| `ROUTE` 常量 | 首页→结果页→报告页跳转 |
| `setLastResult(result)` | 跳转报告页**前**调用，把结果写入缓存 |

**代码中的交互标注**：见 `src/pages/ResultPage.tsx` 顶部注释、`src/store/resultCache.ts` 的 `setLastResult` 文档注释。

### 4.2 R3 → R5（苏和）：提供 UI 组件 + 路由 + 缓存读取

R5 负责 3 个页面 + 历史存储。R3 给 R5 提供：

| R3 提供的接口 | R5 在哪里用 |
|--------------|------------|
| `PageLayout` | 报告页、历史页、Token 页外层容器 |
| `EmptyState` | 历史页空状态、报告页无缓存时 |
| `LevelTag` | 历史页列表项 |
| `ROUTE` 常量 | 报告页/历史页/Token 页跳转 |
| `getLastResult()` | 报告页读取最近一次检测结果，渲染 `result.report` |

**代码中的交互标注**：见 `src/pages/ReportPage.tsx`（已实际调用 `getLastResult()`）、`src/store/resultCache.ts` 的 `getLastResult` 文档注释。

### 4.3 R3 与 R1（张恩）：类型骨架 + 不直接交互

- R3 在 `src/types/index.ts` 里为 R1 维护 `GithubData` / `ApiError` / `RepoInfo` / `FileItem` 的**类型骨架**，所有者标注为 R1。
- R1 实现 `src/api/index.ts`（R3 已放占位 stub）。
- R3 与 R1 无运行时依赖（R3 不调 R1 的函数）；R1 的函数被 R4/R5 调用。
- **localStorage 槽位互不冲突**：R1 用 `opencheck_token`，R3 用 sessionStorage 的 `opencheck_last_result`，R5 用 `opencheck_history`。

### 4.4 R3 与 R2（王靖壹）：类型骨架 + 不直接交互

- R3 在 `src/types/index.ts` 里为 R2 维护 `AnalysisResult` / `CheckItem` / `ScoreInfo` / `Suggestion` 的**类型骨架**，所有者标注为 R2。
